Starting from integer n, find the minimum number of moves to reach m, where each move multiplies the current value by either 2 or 3. It is guaranteed that if a path exists, all paths take the same number of moves. Output the number of moves, or -1 if m is not reachable from n using only these multiplications. The algorithmic approach is divisibility checking: if m % n != 0 return -1, otherwise repeatedly divide m/n by 2 and 3 counting each step; the main challenge is recognising that any remaining factor other than 1 after exhausting 2s and 3s means -1.
