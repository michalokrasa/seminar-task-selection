A string is constructed by three friends in sequence: the first writes one or more 'a's, the second appends one or more 'b's, and the third appends 'c's whose count equals exactly the number of 'a's or the number of 'b's already in the string. Given a string consisting only of 'a', 'b', and 'c', determine whether it could have been formed by this process, checking both the ordering constraint and the count condition. The algorithmic approach is a linear scan to verify the structural order (a+ then b+ then c+) and count each segment; the main challenge is ensuring all three segments are non-empty and that the 'c' count exactly matches one of the other two segment counts.
