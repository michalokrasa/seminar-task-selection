A code obfuscation process replaces variable names in order of first appearance with single letters a, b, c, … (original names are always multi-character). Given a string of lowercase letters representing all identifiers concatenated with whitespace removed, determine whether it is consistent with such an obfuscation. Specifically, the letters must appear in strict alphabetical order of first occurrence — no letter can appear before all preceding letters have been introduced. The algorithmic approach is a single linear scan tracking the set of seen letters and the next expected letter; the main challenge is recognising that the validity condition is purely about first-occurrence order, not about the overall sequence of characters.
